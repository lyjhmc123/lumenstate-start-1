# Design System Rules

## CRITICAL: 컴포넌트 재활용

새로운 컴포넌트를 만들기 전에 반드시 기존 컴포넌트로 대체 가능한지 확인하고, 가능하면 최대한 재활용해야 한다. 불필요한 중복 컴포넌트 생성을 피해야 함.

## CRITICAL: 디자인 토큰 사용

스토리북 Style 카테고리에 명시된 디자인 토큰(색상, 타이포그래피, 간격 등)과 아이콘을 우선 사용해야 한다. 임의의 색상값, 폰트 크기, 간격을 직접 지정하지 말고 theme 토큰을 참조할 것.

### 토큰 사용 규칙

| 카테고리 | 사용 방법 | 예시 |
|---------|----------|------|
| 색상 | theme.palette 토큰 사용 | `primary.main`, `secondary.main`, `grey.100` |
| 타이포그래피 | theme.typography 사용 | `variant="h1"`, `variant="body1"` |
| 간격 | theme.spacing 기반 값 사용 | `p: 2`, `m: 3`, `gap: 1` |
| 아이콘 | lucide-react 라이브러리 우선 사용 | `src/components/style/Icons.stories.jsx` 참고 |

## 중요: MUI 최신 버전 사용

모든 기본 컴포넌트는 MUI의 가장 최신버전을 사용해야 한다.

## 스타일링 규칙

- 모든 컴포넌트의 스타일은 가능한 MUI의 sx 함수를 사용
- 수정시 의존성을 줄만한 기능들을 독립된 컴포넌트로 모듈화
- 새로운 수정, 추가사항이 있을때 지시하지 않은 기존 기능, 형태에 영향을 주지 않도록 조심
- 특별한 의도가 없다면 구글 메터리얼 디자인의 가이드에 기반한 UX에 충실

## 작업 분리 원칙

- **UI 레이어**: 순수 프레젠테이션 컴포넌트 (로직 없음)
- **로직 레이어**: 비즈니스 로직, 상태 관리, API 호출
- Storybook에서는 UI 레이어만 다룸
